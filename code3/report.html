The basic functionality of the e-basket application had to be done
using jsp java beans without scriplet.
The interface is a table where the user has to enter the desired quantity
of 3 items,coffe,sugar and water in order to get the total amount of the price
to be paid.
The whole implementation of the application is done in a jsp file
using an html form where the major jsp javabean commands are embedded.

A JavaBean is just a Java class that adheres to the following rules,
it is a public class
it has a public constructor with no arguments
it has public get and set methods to read and write to properties.

So,for my e-basket i created a two java classes.

One that has setters and getters method for each of the three
user's choices for the quantity.

And the second that sets the property with the value of the user's
input for the quantity multiplied by the price,and also has methods to
get each result and also a method to get the total.

Inside the jsp file i declare and initialize a bean attribute for each
class.I set the property for each one of the quantity-inputs.

So inside my form where i want to have the total result for each item
i do getProperty with name the name of the second class,and i do the same
for the whole Total.

In order for the quantity that the user has entered to be shown after the user
has submitted the form,i put getProperty with name the name of the first class,
in the value of the < input .... /> that every form has.
( < input type="text" .... value="< jsp:getProperty name="Initialbasket" property="qsugar" />"/>

I haven't managed to use cookies cause i couldn't set them without using
scriplet.

I also parse the the price values from an xml file.
